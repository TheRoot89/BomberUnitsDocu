3 How To Add Models
4 ==============
5 
6 Adding custom models requires some creativity, a bit of knowledge in modelling, and the right steps to
7 correctly import the results to Unity.
8 If you do not start in AssetForge, go directly to step 4. Blender will be needed anyway.
9 
10 1 Create a low-poly model in AssetForge (2)
11 -------------------------------------------
12 
13 AssetForge is a good place to start a model from basic low-poly components, if you don't want to model
14 directly in blender and just want some cool results quickly.
15 
16 It is quite self-explanatory. Feel free to add some colors to your surfaces or even textures.
17 > ! Textures are currently not supported nicely in AssetForge 2. We can add them in still later though :)
18 
19 When you are done, safe the model using the FBX export menu entry and choose to not merge the individual
20 blocks.
21 
22 2 Convert the exported model from textual to binary FBX
23 -------------------------------------------------------
24 
25 The AssetForge Export uses FBX ASCII format, which has to be converted to FBX Binary to be used in Blender.
26 Use the "Autodesk FBX Converter x64 2013" for this. You can get this tool for free online, install, select
27 your ASCII file to the left and save your binary FBX to the right. Be sure to select 'binary' and click
28 'convert'.
29 
30 3 Import the FBXmodel in Blender
31 -----------------------------
32 
33 From any Blender project go to File -> Import -> Choose the format you need to import. This is FBX if you
34 exported from Asset Forge earlier.
35 
36 4 Refine the model in blender
37 -----------------------------
38 
39 Most importantly, you should try here to simplify the geometry and merge parts that should move together.
40 You may want to create a second 'destroyed' version of your model as well. This should be composed of a
41 collection of separate objects, which represent pieces of the original model to be thrown by an explosion.
42 
43 Merging multiple objects together while getting rid of the internal 'additional geometry' is a bit tricky.
44 Get Toni to show this to you, preferably offering cake.
45 
46 While having selected faces in Edit Mode, you can select the material to be rendered for these faces to the
47 right in the materials tab and list. You have to confirm with "Assign". This will be imported to Unity
48 correctly to color your model with the materials you set.
49 
50 Other things to do are:
51 
52 * Arrange the Meshes in a reasonable object hierarchy
53 * Check the scaling fits to Unity: 1m in Blender is equivalent to 1 unit (as per default)
54 * Apply all transforms
55 * Set the origin of the object (normally center of mass for correct physics)
56 
57 5 Import the blender model in Unity
58 -----------------------------------
59 
60 Unity can read blender models directly. Save in blender and copy or move that file to the Assets/Models
61 folder. Back in Unity, you can set some import options, especially exporting materials from the model if you
62 want to manipulate these further directly in Unity.
63 
64 6 Create a prefab to spawn the model
65 ------------------------------------
66 
67 1. Drag the imported model to a scene (to make an instance of it).
68 2. Right-click -> Unpack Prefab for this object to sever the connection to the imported model.
69 3. Drag the object to the folder structure, to create a new prefab. These go to the Assets/Prefabs folder.
70 4. You can continue to customize your prefab as needed. Examples:
71 
72  * Placing the model underneath a new prefab root parent and fixing the relative positioning.
73  * Adding more components
74  * Removing camera and light objects that came from blender.
75  * Modifying the extracted materials to get the looks you like in Unity (a bit different to Blender looks).
