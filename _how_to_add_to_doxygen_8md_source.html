3 How to add to our *Doxygen* docu
4 ================================
5 
6 In general
7 ----------
8 
9 The docu consists of auto-generated content from the parsed scripts and additional pages on various topics.
10 To support the duty is good practice as a developer, so:
11 
12 * *MUST*: Comment your code with Doxygen-compatible syntax ( -> have a look at existing code).
13 * *OPTIONAL*: Create your own so-called 'Related pages' (this is one of them) to describe overall concepts or
14  details you don't want to explain in your class documentation via above-mentioned comments.
15 
16 If you updated the master branch with new documentation, think about pushing the generated html files to
17 the root of the repo hosting the docu [here](https://github.com/TheRoot89/BomberUnitsDocu).
18 
19 > **Note:** We are currently using Doxygen Version 1.8.13 for Windows. Other versions may have queer effects
20 > as each version tends to have their own quirks...
21 
22 Documentation structure
23 -----------------------
24 
25 Regarding the project (game feature) content itself, there are distinct steps from an idea, to its
26 evaluation/integration on system level, to its realization on SW level. This is explained on the
27 \ref page_concepts pages.<br>
28 The resulting SW has to be documented as well, which is done in the \ref page_sw_architecture and the
29 generated SW detailed design (API). Additionally, there should be pages on development topics, such as tooling
30 and How-Tos.
31 
32 Modifying a page
33 ----------------
34 
35 You can change the Markdown content with any text editor. Some editors, like VS Code, are able to show
36 rendered previews of the Markdown, which gives a hint for how it's going to look as Doxygen output as well.
37 
38 When adding or modifying content of pages, consider to build the docu afterwards locally, to check whether the
39 result matches your expectations. Also consider uploading the HTML output (in *Doc/doxy_out*) to our Docu
40 repository: [https://github.com/TheRoot89/BomberUnitsDocu](https://github.com/TheRoot89/BomberUnitsDocu)<br>
41 You do not have to create a branch or anything, simply commit your new docu generated content to the master.
42 
43 Creating a new page
44 -------------------
45 
46 Ok, so assuming you want to explain a concept, do a How-to or whatever comes to your mind - and you need to
47 add a new page:
48 
49 1. Create a new .md file and add it somewhere reasonable in ```Doc/doxy_in/docs_doxygen```.<br>
50  Should you want to check in any files Doxygen *cannot parse* but you for example want to provide a link to
51  it, add those to ```Doc/doxy_in/docs_non_doxygen```.
52 2. Give the file a header on the first line like this: <b>\\page [page_tag] [page title]</b>
53 3. Add some content to the page, typically using the double-underline to mark the main title.
54 4. Add your page to the navigation tree by making it the child of a reasonable parent page.
55  This is done by adding the <b>\\subpage [page_tag]</b> command somewhere on the parent page.<br>
56  You have to make your page a child of another, otherwise the new page will appear on the top level of the
57  navigation tree to the left.
58 5. Add references to your page from other pages or from code if you like: <b>\\ref [page_tag]</b>
59 
60 At the positions of references and \\subpage commands Doxygen will place the page title (may contain spaces)
61 you also defined on the first line.
62 
63 > Note: In case you are reading this as a text file: the commands only use a single backslash: "\". The second
64 > one is an escape character for writing this without invoking the command.

66 Using PlantUML graphs in Markdown files
67 ---------------------------------------
68 
69 PlantUML provides a syntax to specify UML graphs inline from Markdown. The result is rendered here in the
70 preview and in Doxygen. Writing PlantUML with the following Syntax will generate it in the VSCode preview, but
71 also show the initial, quite ugly code block ticks in Doxygen. This is unfortunate but I found no workaround
72 yet. Also the set style configuration file is not included when previewing.
73 
74 An example for a PlantUML activity diagram:
75 
76 ~~~plantuml
77 \startuml
78  Sender->Receiver : Command()
79  Sender<--Receiver : Ack()
80 \enduml
81 ~~~

83 Using images in Doxygen
84 -----------------------
85 
86 To import an image file use the command \!\[Caption text](/path/to/img.jpg). Again, without any backslashes,
87 should you be reading this in plain text.
88 
89 Your have to give the path relative to the Doc folder as this is the execution directory for Doxygen.
90 Also, you have to place your images in the ```doxy_in/images``` folder, so your path will be:
91 ```/doxy_in/images/[image_file]```
92 
93 ![Example image](/doxy_in/images/Dunny.png)
94 
95 > Please note images will usually not be found and displayed in the markdown preview.
