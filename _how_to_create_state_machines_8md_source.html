3 How to create State Machines (using the Unity Animator System)
4 ==============================================================
5 
6 In general
7 ----------
8 
9 ![Example state machine](/doxy_in/images/State_machine_example.png)
10 
11 * A state machine is used to represent the states a particular object or system may be in, with one being
12  the current state at any given time.<br>
13  Examples: a simple enemy AI, menu page transitions or states of the game.
14 * Taken together, the set of states, the set of transitions between them and the variable to remember the
15  current state form the state machine.
16 * In Unity, you can create an asset called an Animator Controller. This is a state machine template normally
17  used for transition between different object or character animations.
18 * The state machine states are usually associated to animations, but this is actually optional if you want to
19  create a pure-logic state machine.
20 * To run your state machine, add a component called Animator to a GameObject and set it up with any Animator
21  Controller that you’ve created. This is now an instance of your state machine.
22 * To run logic on each state, you need to create a script derived from the class StateMachineBehaviour.
23  This script can be added to a state in the state machine (Animator) view.
24 
25 StateMachineBehaviour
26 ---------------------
27 
28 * Use ```OnStateEnter()``` and ```OnStateExit()``` methods to e.g. start and stop listening to game events
29  when the state is active.
30 * To clean up the current state properly, you’ll need to make sure to handle ```OnDisable()``` as well, as
31  the above callbacks are not called if the object or Animator component is disabled.
32 * You can create a library of StateMachineBehaviour subclasses to reuse in your state machines.
33 
34 Settings it up
35 --------------
36 
37 1. Create a new "Animator Controller" from the right-click menu. A double-click on it will bring you to the
38  Animator Editor view, showing the first state boxes: Start and Exit.
39 2. Create the state boxes you need and draw the desired connections using right-click->make transition.
40 3. For each transition, set transition duration to zero and deactivate the exit time. If transition duration
41  is non-zero, the next state’s OnEnter will be called before the current state’s OnExit and both are active
42  at the same time! (This is the intended function of the transition between to animation states: a blend-
43  over between the two.)
44 4. Add transition Triggers in the parameter sidebar and assign them as conditions to the transitions you
45  created. A logical 'Or' is done by adding more than one transition between the same source and target.
46  A logical 'And' is done by having multiple conditions on a single transition.
47 5. Add your Controller to an Animator component of a suitable object of your choice. Example: A GameState
48  state machine would go to the 'GameManager' persistent object.
49 6. Get the Animator component, which represents the State Machine from your script. You may want to wrap this
50  in your own StateMachine type class.
51 7. Call ```SetTrigger("Trigger name")``` from code on that controller object to enable the transitions.
52 8. Add subclasses of StateMachineBehaviours to individual states of the machine. As explained above, these
53  possess Callbacks, which are called from Unity when the relevant state is entered or exited.
54  This mechanism works very much like the 'normal' MonoBehaviours you are probably used to.
55 
56 Notes
57 -----
58 
59 * Do not trigger transitions more than once, because they are only consumed once per frame!
60 * Transitions using triggers will advance to the next state in the same frame.
61  Transitions based on other parameter types take an extra frame.
