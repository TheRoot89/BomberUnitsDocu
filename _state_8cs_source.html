5 /// Extended functionality for Unity SM Behaviours and makes them complete 'States'.
6 /// Only one such state is currently allowed per State Box in the Animator Controller!
7 ///
8 /// This is intended to provide safer transitions by enforcing checks for whether a transition is
9 /// allowed for a certain state. This prevents carry-over or false trigger flags to the next states.
10 /// To do this:
11 /// * state name and allowed triggers are listed when in the SM editor
12 /// * These are checked when a state transition is requested.
13 /// * Only the currently active state can be the source for triggering a transition.
14 ///
15 /// The base classes offer for inherited behaviours:
16 /// * Awake() callback
17 /// * OnMyStateEnter() callback
18 /// * OnStateExit() callback
19 /// * OnStateUpdate() callback
