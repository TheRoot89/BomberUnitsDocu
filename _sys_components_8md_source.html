3 System Components
4 =================
5 
6 This page gives an overview of the currently implemented Scripts. The SW architecture realizes the game
7 concepts in a hopefully understandable way. See \ref page_concepts
8 
9 ```plantuml
10 @startuml
11 
12 component GameManager [
13  **GameManager**
14  <persistent, has state>
15 ]
16 
17 component UIManager [
18  **UIManager**
19  <persistent, has state>
20 ]
21 
22 component MatchManager [
23  **MatchManager**
24  <created for each match>
25 ]
26 
27 component PlayerManager [
28  **PlayerManager**
29  <persistent>
30 ]
31 
32 component NetManager [
33  **NetManager**
34  <persistent>
35 ]
36 
37 GameManager --> UIManager
38 GameManager --> MatchManager : Holds references
39 GameManager --> PlayerManager
40 GameManager --> NetManager
41 @enduml
42 ```
43 
44 Each of these behaviors is attached to an invisible GameObject. They represent the typical tasks which need to
45 managed by the program.
46 Please find information on the individual managers in their source comments or the
47 detailed description section in their Doxygen docu.
48 
49 | Component | Short description |
50 |-------------------------------|-------------------------------------------------------|
51 | BomberUnits::GameManager | Manage overall game state and organize interaction |
52 | BomberUnits::UIManager | Manage UI inputs and delegate reactions accordingly |
53 | BomberUnits::MatchManager | Initialize and manage a single match's state |
54 | BomberUnits::PlayerManager | Manage local and remote players |
55 | BomberUnits::NetManager | Manage Network interactions with Photon for MP |

57 Custom Classes
58 --------------
59 Additionally, there is a range of custom classes which don't need to be behaviors. Most importantly:
60 
61 * Container types, such as BomberUnits::Player or BomberUnits::InputScheme
62 * UI classes for managing UI components, grouping and animation. See BomberUnits::UIMenuPage
63 
64 Other Scripts
65 -------------
66 Categorized into folders we also have scripts for:
67 
68 * *Effects*, e.g. of explosions, throwing and breaking objects
69 * *Game Mechanics*, e.g. player controller and loot spawner
70 * *UI behavior*, e.g. billboards or the logics of certain dialogs
71 * *Misc*, for other topics, such reading and writing from/to files.
