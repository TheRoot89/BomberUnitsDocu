3 Tooling
4 =======
5 
6 Every developer collects the tooling he finds works best for his tasks, so here are our recommendations to use
7 for this project. The list does not claim to be complete and is not binding of course :)
8 
9 Unity
10 -----
11 Best use [UnityHub](https://unity3d.com/de/get-unity/download) to manage the ever-changing versions of Unity
12 and your installations thereof. You can make sure the right version is associated to this project when firing
13 it up.
14 
15 > Please find the correct Unity version on the \ref index!
16 
17 For more introductory words on Unity, have a look at the "On Unity" section at: \ref page_intro.
18 Unity provides a lot of tutorials on their own page, as well as on.
19 
20 VS Code
21 -------
22 You may choose to install and use Visual Studio together with Unity for a full-blown IDE.
23 However the more lightweight VS Code is a great and lean alternative!
24 
25 Have a look here for how to set Visual Studio Code up: \subpage page_vs_code_setup

27 Git
28 ------
29 We use Git for versioning and GitHub to track issues, ideas and implementation status. You may want to
30 download the [GitHub Desktop](https://desktop.github.com/) client for ease of use with this repo.
31 
32 GitHub
33 ------
34 Explore our code repo hoster yourself a bit :) <br>
35 The most prominent feature we use is the project and issue handling. We can sort ideas and features into
36 projects and status columns, convert these to issues, and collaborate on them. This allows for easy team
37 organization.

39 Doxygen
40 -------
41 Doxygen is the defacto-standard for creating automated code documentation. The auto-generated parts combine
42 nicely with manually created pages, such as this one. The configuration can be a bit fiddly, but you don't
43 have to care for this too much if everything is set up correctly.
44 
45 > Please use Version 1.8.13 (Win) to avoid any issues though! Install it from the Internet :)
46 
47 It parses a given set of input files and folders in search for compatible text and code files. It then
48 extracts the code symbols and attaches correctly written code comments as description of these symbols.
49 Found text pages are converted to HTML pages.<br>
50 It then creates a website structure from which all found pages and all code symbols (the API) can be
51 navigated.
52 
53 In order to support the auto-generation, **please add your code comments in the Doxygen format**. Have a look
54 at existing code comments to see how this looks. Most importantly, one can describe a code symbol at its place
55 of declaration, together with parameters and return value for functions.
56 
57 The Doxygen output html is **hosted on GitHub** as well, by uploading it as the content of a repo:<br>
58 [https://theroot89.github.io/BomberUnitsDocu/index.html](https://theroot89.github.io/BomberUnitsDocu/index.html)
59

65 Blender
66 -------
67 Blender is a 3D modelling tool. We can use it to create models, textures and animations. Learning Blender is a
68 whole topic of its own. Importantly, we can open and save most common model formats with it and Unity even
69 supports direct import of blender files.
70 
71 AssetForge
72 ----------
73 A simple tool to create low-poly 3D-Models. These can be used as a style of their own or be used as
74 placeholders. Saves to .obj files, which have to be imported as assets. This tool is not free however!
