3 The UI Menu Page System
4 =================================
5 
6 System Requirements
7 -------------------
8 
9 The menu page system is intended to provide animations for menu components as well as grouping, but still
10 allow to use the Unity-inherent Layout features etc.
11 
12 The classic approach is having multiple full menu "pages" readily set up with all their children on the canvas
13 at the same time. These would then be made visible or invisible as pages are switched. We instead want to be
14 able to animate and pan into view the individual elements (often panels with buttons and images)
15 independently. By parenting, also complex UI elements can be animated with this of course.
16 
17 The concrete menus may be found in the GDD under the Doc folder or in the Game Design section once written :)
18 
19 System Solution
20 ---------------
21 
22 Unity provides its own Animation system, which we also use to realize State Machines. Have a look here for
23 details: \ref page_how_to_animate .
24 
25 This animation system is one, quite flexible approach handle movement and appearance of UI elements, as
26 opposed to hard-coding the element movements.
27 
28 In addition, for the transition of pages, another State machine is used.
29 
30 SW Requirements
31 ---------------
32 
33 The system should be independent of the specific project. One should be able to set up an arbitrary menu
34 without having to change the code.
35

37 -----------
38

40
