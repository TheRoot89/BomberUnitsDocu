3 How to set up VSCode nicely
4 ===========================
5 
6 Visual Studio Code is a lightweight Editor and IDE not dependent on specifying full *projects* or *solutions*
7 as for the full Visual Studio. It is quite fast and highly customizable through the use of *Extensions*, see
8 recommended extensions below.
9 
10 Setting up VSCode for Unity
11 ---------------------------
12 Follow [this site](https://code.visualstudio.com/docs/other/unity). The steps are in short:
13 
14 1. Download and install .NetCore (use version compatible to Visual Studio 2017). Restart.
15 2. Install the C# extension.
16 3. Open the Unity project.
17 4. Set Visual Studio Code as the default External Script Editor.
18 5. Extend VSCode with further extensions, like Debugging for Unity, as listed on the website above after the
19  setup section.
20 6. Enable code completion.
21 
22 **Note:** you may have to delete all the .sln and .csproj files in the root folder to have Unity build the
23 configuration specialized for VSCode.
24 
25 Recommended Extensions and their Settings
26 -----------------------------------------
27 
28 To install any of these, search for their name on the *Extensions* sidebar and click *Install*.
29 
30 ### C/C++
31 Code highlighting and other goodies when working with C or C++. Must-have!
32 
33 ### CSharp
34 Use this if working for example with Unity C# code. (Could not use the 'Hash' character in the header. The
35 extension is actually called *C#* .)
36 It also provides a formatter to work with our .cs sources. Go to the settings for this extension and check
37 whether the formatter is enabled (Format: Enable). The formatter uses the settings in 'omnisharp.json' which
38 is placed in the repo root folder.
39 
40 ### Regions with Colors
41 Provides region highlight for C# code (see sources for examples). You can set the color with alpha channel in
42 the settings. I use purple: rgba(132, 27, 237, 0.1)
43 
44 ### Git Lens
45 This adds way more functionality, such as Diffs, History, Blame annotations etc. to your source control in VS
46 Code.
47 You may want to configure it to show everything under the normal *Source Control* sidebar, so it does not
48 generate an additional icon to the side.
49 
50 ### Git Graph
51 Allows you to show a Git Graph (or *Git log*) view from the *Source Control* side bar via a small additional
52 icon at the top, or via the "Git Graph" button in the status bar.<br>
53 It is way easier to get an overview on what is going on using this visual representation of branches etc..
54 
55 ### Bracket Pair Colorizer 2
56 This will highlight cascaded brackets such as ([{}]) in your code, making your life a bit easier :)
57 
58 ### Code Spell Checker
59 This is a nice spelling checker. You can disable certain file types and it will recognize CamelCase and the
60 like as a concatenation of words. You may have to add a lot of words to your *User Dictionary* in the
61 beginning though.
62 
63 Hint: When having this and other kinds of checks, such as Unused-include checks (C# support), use a filter of
64 `!Photon` in the bottom *PROBLEMS* panel to not display findings from the Photon sources we use.
65 
66 ### PlantUML
67 Provides previews of plantuml code blocks in Markdown or own PlantUML files when viewing these with VS Code.
68 Be sure to set the mode to *Server*. This documentation should contain some examples of how to write PlantUML
69 diagrams in the Markdowns. Click the top right "Open Preview to the Side" button to have your Markdown
70 rendered.
71 
72 ### Markdown Preview Github Styling
73 Makes your markdown background white and is much closer to the usually rendered result this way! Also UML
74 diagrams with black text will be readable now ;)
75 
76 ### TODO Highlight
77 ...lets you highlight tags you can set yourself. Typical comment flags, such as //TODO are pre-set. You may
78 want to add the Doxygen-specific flag \\todo though. You can also define your own, such as "//TONI"
79 
80 ### vscode-icons
81 Nice icons for your folders and files, increasing explorer view usability a lot!
82 
83 ### Vim
84 If you are that type of person, you can have Vim input options for VS Code with this extension.
85 When going for Vim, I highly recommend to also use a re-mapper to have ctrl and escape set to your Caps :)
86 
87 Link: [Dual Key Remap on GitHub](https://github.com/ililim/dual-key-remap)
