3 How To Add Cool Holo UI and Occlusion Hints
4 ===========================================
5 
6 The main target of the below is to not have to set individual materials per sprite and to allow for Holo UI
7 to be rendered in front of any other geometry.
8 
9 For this there are two options, both differentiate objects based on the Layer they set to be on:
10 
11 * A Camera stack: The second (overlay) camera has to follow the first camera and renders selected layers on
12  top.
13 * Separation of renderers in the renderer Stack, with some layers being taken care of by a separate renderer
14  (or so-called or renderer feature). This feature has somewhat more power than the camera approach.
15  Also an override material can be defined, which ignores the material set to the objects.
16 
17 The decision for now is to use the camera stack for the UI overlay, because it is simpler for now to extend.
18 For this, the additional camera ist set up to follow the zoom level of its main camera.
19 You only need to add UI objects to the UI layer to have them rendered before anything else.
20 Be sure though to use the mainCamera prefab in your scenes, as this contains the additional cam and settings.
21 
22 The Renderer Features are used to as shown in this video here: [https://www.youtube.com/watch?v=szsWx9IQVDI]
23 To add more stuff to be rendered when occluded, add them to the layers defined as 'Mask' in the renderer
24 feature. Please not that an overwrite material is currently set there. Duplicate it and change the settings
25 for to change occlusion appearance. Currently the same material is used for both effects explained here.
