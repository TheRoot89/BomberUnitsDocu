3 How to maintain the CI (GitHub workflows)
4 =========================================
5 
6 GitHub Workflows allows to run certain commands, e.g. for Pull Requests.
7 We currently test whether the PR builds successfully. Here is how to set this up.
8 
9 The workflows scripts (configuration files) can be found under "repo_root".github/workflows/*.yml
10 
11 1 Check you have set the correct Unity versions
12 -----------------------------------------------
13 
14 Check the workflow scripts for mentions of the Unity version to use and update it to what is used currently.
15 
16 2 Run the activate.yml workflow
17 -------------------------------
18 
19 When opening a PR, the workflows should run automatically. Open the Activation workflow and download the
20 saved Activation .alf file, which should have been stored as a build artifact after execution.
21 
22 3 Acquire the corresponding .ulf file
23 -------------------------------------
24 
25 Got to Unity here: [https://license.unity3d.com/manual]()
26 Upload the .alf and get the .ulf. Both should be named after the Unity version you are using.
27 
28 4 Upload the content of the .ulf as UNITY_LICENSE secret
29 --------------------------------------------
30 
31 Set up or update the GitHub secret "UNITY_LICENSE" with all the contents of the .ulf.
32 Secrets can be found in the repository settings.
33 
34 5 Run the build workflow
35 ------------------------
36 
37 Now the build workflow (buildMaster.yml) should build successfully using the uploaded license!
38 
39 Troubleshootings
40 -----------------
41 
42 We hope the storage space is sufficient, but it may happen you have to change the settings in the "Remove
43 artifacts" step to remove more old artifacts.
