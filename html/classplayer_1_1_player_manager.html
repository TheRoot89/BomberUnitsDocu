Detailed Description

The PlayerManager controlls the local and remote players.

It is a Singleton and persistent over scenes, like the GameManager. This manager also provides Input Schemes for the local players and interacts with the UIManager to provide UIPlayerSlots.

Tries to read from preferences to create the initial "main" player.

Assigns default values if the keys are not found in the preferences. This is to be triggered during initialization by the GameManager. The function assigns the player to a slot and opens another slot if more players are allowed.

◆ RemovePlayer()

void RemovePlayer ( Player  p)
inlineprivate

Player removal can be caused by:

  • a user action: remove the player for which deletion was called from the UI. This can happen when this or a remote client removes a player and is transmitted via an event.
  • or disconnect of this client: RemoveAllRemotePlayers of any clients which were present before.
  • or disconnect another client: RemoveAllRemotePlayersOf the particular client.
