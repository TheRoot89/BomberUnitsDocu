BomberUnits

BomberUnits is a Bomberman clone in Unity as a learning project. It is developed in 3D and supports mixed local and remote multiplayer via Photon.

Last "stable" release:

Unity version to use: 2018.2.21f1
How to start: Check out the correct commit, install and assign the correct Unity version to the project in UnityHub, load the project, load and play the "MainMenuScene".

About the Documentation

The docu shall be built up like this:

~~~plantuml

~~~

The docu consists of auto-generated content from the parsed scripts and additional pages on various topics. Most importantly, the Concepts pages each explain: (See: Concepts)

  • System Requirements: A typical problem a game program such as this poses, which needs to be solved.
    In essence, these are the results of the Game Design process, i.e. what we want to achieve. The GameDesign can currently be found in a ppt file.
  • System solution: The approach taken to solve the problem from a top-level perspective.
  • SW requirements (in part): Constraints on how to implement the system solution best.
  • SW solution: Explanations on how the solution works if solved by multiple classes. Also find usage information here for how the implementation is to be used.
    The actual SW solution detailed design is represented by the code comments in the implementation files. This is parsed and accessible at the Doxygen output for each of the related classes.

These documentation pages are viewable as text files, rendered as Markdown, e.g. in VSCode preview, or after generating the Doxygen docu HTML as explained below.

The Doxygen-specific "\\ref" links and commands will not work when viewing the files directly or as Markdown, but you may search textually for the referenced unique keywords to find the relevant content.

Building the Documentation

Prerequisites:
You may have to install Doxygen and GraphViz (dot) from the internet. Google is your friend here.

Use the PowerShell script build_docu.ps1 in the Doc folder to build the documentation. It calls doxygen Doxyfile to call doxygen with its configuration and creates a link to the start page.
Double-click it to execute and select PowerShell as program to open it with if necessary. In order to allow PowerShell script execution you may have to execute the following in a PowerShell you started with Admin rights: Set-ExecutionPolicy Unrestricted

Adding to the Documentation

  • MUST: Comment your code with Doxygen-compatible syntax ( -> have a look at existing code).
  • OPTIONAL: Create your own so-called 'Related pages' (this is one of them) to describe overall concepts or details you don't want to explain in your class documentation via above-mentioned comments. See How to Doxygen for more details.
