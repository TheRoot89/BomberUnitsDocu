System Components

This page gives an overview of the currently implemented Scripts. The SW architecture realizes the game concepts in a hopefully understandable way. See Concepts

```plantuml

```

Each of these behaviors is attached to an invisible GameObject. They represent the typical tasks which need to managed by the program. Please find information on the individual managers in their source comments or the detailed description section in their Doxygen docu.

Component Short description
BomberUnits::GameManager Manage overall game state and organize interaction
BomberUnits::UIManager Manage UI inputs and delegate reactions accordingly
BomberUnits::MatchManager Initialize and manage a single match's state
BomberUnits::PlayerManager Manage local and remote players
BomberUnits::NetManager Manage Network interactions with Photon for MP

Custom Classes

Additionally, there is a range of custom classes which don't need to be behaviors. Most importantly:

  • Container types, such as BomberUnits::Player or BomberUnits::InputScheme
  • UI classes for managing UI components, grouping and animation. See BomberUnits::UIMenuPage

Other Scripts

Categorized into folders we also have scripts for:

  • Effects, e.g. of explosions, throwing and breaking objects
  • Game Mechanics, e.g. player controller and loot spawner
  • UI behavior, e.g. billboards or the logics of certain dialogs
  • Misc, for other topics, such reading and writing from/to files.
