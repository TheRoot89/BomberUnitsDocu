Tooling

Every developer collects the tooling he finds works best for his tasks, so here are our recommendations to use for this project. The list does not claim to be complete and is not binding of course :)

Unity

Best use UnityHub to manage the ever-changing versions of Unity and your installations thereof. You can make sure the right version is associated to this project when firing it up.

Please find the correct Unity version on the Home!

For more introductory words on Unity, have a look at the "On Unity" section at: Introduction. Unity provides a lot of tutorials on their own page, as well as on.

VS Code

You may choose to install and use Visual Studio together with Unity for a full-blown IDE. However the more lightweight VS Code is a great and lean alternative!

Have a look here for how to set Visual Studio Code up: VSCode Setup

Git

We use Git for versioning and GitHub to track issues, ideas and implementation status. You may want to download the GitHub Desktop client for ease of use with this repo.

GitHub

Explore our code repo hoster yourself a bit :)
The most prominent feature we use is the project and issue handling. We can sort ideas and features into projects and status columns, convert these to issues, and collaborate on them. This allows for easy team organization.

Doxygen

Doxygen is the defacto-standard for creating automated code documentation. The auto-generated parts combine nicely with manually created pages, such as this one. The configuration can be a bit fiddly, but you don't have to care for this too much if everything is set up correctly.

Please use Version 1.8.13 (Win) to avoid any issues though! Install it from the Internet :)

It parses a given set of input files and folders in search for compatible text and code files. It then extracts the code symbols and attaches correctly written code comments as description of these symbols. Found text pages are converted to HTML pages.
It then creates a website structure from which all found pages and all code symbols (the API) can be navigated.

In order to support the auto-generation, please add your code comments in the Doxygen format. Have a look at existing code comments to see how this looks. Most importantly, one can describe a code symbol at its place of declaration, together with parameters and return value for functions.

The Doxygen output html is hosted on GitHub as well, by uploading it as the content of a repo:
https://theroot89.github.io/BomberUnitsDocu/index.html

There are some How-tos to help you further in using this tool:

  • How to add to Doxygen
  • How to generate Doxygen

Blender

Blender is a 3D modelling tool. We can use it to create models, textures and animations. Learning Blender is a whole topic of its own. Importantly, we can open and save most common model formats with it and Unity even supports direct import of blender files.

AssetForge

A simple tool to create low-poly 3D-Models. These can be used as a style of their own or be used as placeholders. Saves to .obj files, which have to be imported as assets. This tool is not free however!
