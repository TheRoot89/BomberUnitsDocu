Tooling

Every developer collects the tooling he finds works best for his tasks, so here are our recommendations to use for this project :)

Unity

Best use UnityHub to manage the ever-changing versions of Unity and your installations thereof. You can also make sure the right one is associated to this project when firing it up.

Please find the correct Unity version on the mainpage!

For more introductory words on Unity, look at the "On Unity" section at: Introduction.

VS Code

You may choose to install and use Visual Studio together with Unity for a full-blown IDE. However the more lightweight VS Code is a great and lean alternative! Have a look here for how to set Visual Studio Code up: VSCode Setup

Git

We use Git for versioning and GitHub to track issues, ideas and implementation status. You may want to download the desktop client for ease of use with this repo.

Doxygen

Doxygen is the defacto-standard for creating automated code documentation. The auto-generated parts combine nicely with manually created pages, such as this one. The configuration can be a bit fiddly, but you don't have to care for this too much if everything is set up correctly.

In order to support the auto-generation, please add your code comments in the Doxygen format. Have a look at existing code comments to see how this looks.

For adding custom pages, look here for more info: How to Doxygen

Blender

Blender is a 3D modelling tool. We can use it to create models and animations. Learning Blender is a whole topic of its own. Importantly, we can open and save most common model formats with it and Unity even supports direct import of blender files.

AssetForge

A simple tool to create low-poly 3D-Models. These can be used as a style of their own or be used as placeholders. Saves to .obj files.
